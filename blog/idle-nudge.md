# Idle Nudge——别闲着，回去写代码

> DeskPet 新功能：长时间不用 Claude，桌宠会主动「催你」回去干活。

---

## 它是什么

当 session 用量百分比连续 5 分钟没有变化——也就是你一直没在用
Claude——设备会自动弹出一个全屏覆盖层：

![Idle Nudge 覆盖层](images/nudge.png)

左边是经典鼠标指针手型，不停地左右晃动；右边是一只正在敲代码的
Clawd 像素动画。意思很明确：**点我，回去干活。**

轻触屏幕，设备通过 BLE 向主机发送指令，自动打开 Claude 应用，
覆盖层随即消失。如果你继续偷懒，它还会再弹出来——直到 session
数据真正发生变化才会停止催促。

## 怎么做到的

### 经典鼠标手型

手型是经典 cursor hand 的 90° 旋转版——16×11 像素网格，以 10 倍
放大渲染到 LVGL canvas 上。白色填充、灰色描边，在 AMOLED 黑底上
非常清晰。

晃动效果用 LVGL 内置动画引擎实现：±10px 水平往复，700ms 一个
周期，制造「戳一戳」的视觉暗示。

### Clawd 敲代码动画

动画复用了 splash 模块的 `work coding` 动画组。为了让 nudge 覆盖层
能独立渲染，给 splash 模块新增了 `splash_render_scaled()` 接口——
以 12 倍放大渲染到独立的 240×240 canvas，不干扰主 splash 的全屏
渲染。

### 点击即跳转

轻触覆盖层，固件通过 BLE 向主机发送 `0x20` 指令。守护进程收到后
调用 `open -a Claude`（macOS）或 `xdg-open`（Linux）打开 Claude
应用，覆盖层随即消失。

### 循环触发

覆盖层被点掉后重新计时。再次空闲够久会再弹出，直到 session 数据
真正发生变化才停止催促。判断逻辑分三条路径：

1. **用量变了** → 重置计时器，关掉覆盖层
2. **刚被点掉** → 重新开始倒计时
3. **倒计时到了** → 弹出覆盖层

## 未来功能展望

受 [Taby](https://heytaby.com/)（一款贴在显示器边框上的桌面机器人
助手）的启发，结合 DeskPet 自身的硬件能力，规划了以下方向。每条
都附了可行性判断——毕竟我们的硬件是一块 ESP32-S3 + 480×480 AMOLED,
不是通用计算机。

### 可直接做（硬件已就绪）

- **用量趋势图**——新增一屏，用 sparkline 画出当天 session 用量的
  变化曲线，一眼看出什么时段烧得猛。守护进程每 60 秒轮询一次，
  把历史数据点推到 ESP32，LVGL 有现成的 chart 控件。
  *可行性：高。纯软件，数据链路已通。*

- **触顶警报**——当 status 变成 `limited` 时，全屏弹一只「沮丧」的
  Clawd 像素动画，比干巴巴的百分比更直观。
  *可行性：高。复用 splash 动画系统，加一个状态判断即可。*

- **用量焦虑表情**——splash 动画按用量强度切换情绪：低用量悠闲打盹，
  高用量换成更「慌」的动画。类似 Taby 的 50+ 表情对用户行为的响应。
  *可行性：高。分组轮换机制已有，只需调整分组映射规则。*

- **摇一摇 / 敲桌互动**——利用板载 QMI8658 加速度计检测物理动作，
  触发特殊动画或彩蛋。类似 Taby 的滑动/点击反馈，但用的是 IMU
  而不是触摸手势。
  *可行性：高。IMU 驱动已接好，加阈值检测即可。*

- **番茄钟 / 专注计时**——类似 Taby 的 Pomodoro 计时器，在设备上
  显示 25 分钟倒计时，结束后 Clawd 播放庆祝动画。可以用物理按键
  启停，触摸屏切换工作/休息模式。
  *可行性：高。倒计时屏的代码可以复用，UI 框架已成熟。*

- **活动洞察——编码时长统计**——守护进程监测 Claude Code 进程的
  活跃状态，推送每日编码时长分布到设备。Taby 用的是全局应用时间
  分配，我们聚焦在 Claude Code 本身。
  *可行性：中高。设备端简单（接收+渲染），主要工作在守护进程端
  增加进程监控逻辑。*

### 需要守护进程配合（中等工作量）

- **Workspace 一键启动**——类似 Taby 的 workspace launcher，长按
  物理按键让守护进程同时打开一组应用（终端、编辑器、浏览器等）。
  BLE 指令协议已有 `0x20`（打开 Claude）的先例，扩展即可。
  *可行性：中。协议扩展简单，但需要用户自定义应用列表的配置机制。*

- **每日习惯打卡**——Taby 的 streak 追踪功能。每天是否用了 Claude
  Code？连续天数？设备上显示日历点阵或火焰图标。数据由守护进程
  从 API 历史推算。
  *可行性：中。需要守护进程持久化历史数据，设备端渲染较轻量。*

- **AI 对话增强**——当前已有 DeepSeek 驱动的 preset 问答屏。可以
  扩展为自由文本输入（通过守护进程的桌面端输入框中转），让 Clawd
  变成一个微型聊天终端。类似 Taby 规划的 AI 语音交互。
  *可行性：中。文本中转可行，但 480×480 屏幕上的输入体验有限，
  更适合做「快捷问答」而非完整聊天。*

### 需要额外硬件（长期方向）

- **语音交互**——Taby 计划通过板载麦克风实现语音指令。ESP32-S3
  支持 I2S 数字麦克风，但 Waveshare 2.16" 板没有板载麦克风，需要
  外接 INMP441 或类似模块。语音识别需要云端（ESP32 算力不够跑
  本地 ASR）。
  *可行性：低。需要硬件改造 + 云端 ASR 服务，复杂度跳了一个量级。*

- **环境光自适应**——Taby 计划加光传感器。Waveshare 板没有，需要
  外接 BH1750 等。好处是根据环境亮度调 AMOLED 亮度省电护眼。
  *可行性：低。I2C 总线有空位，但需要飞线焊接，不适合普通用户。*

- **独立供电（电池）**——目前靠 USB 供电。板子有 AXP2101 PMU，
  理论上可以接锂电池，但需要外壳改造。Taby 也面临同样问题（电池
  模块尚未发货）。
  *可行性：中低。PMU 硬件支持，但需要结构设计和安全认证。*
